# 학생 등록 및 점수 입력

선생님이 학생을 등록하거나, 기존 학생의 퀴즈 결과를 기록합니다.
데이터는 `students/` 디렉토리에 학생별 JSON 파일로 저장됩니다.

## 인수 파싱

`$ARGUMENTS`를 공백으로 분리하여 파싱하세요.

| 위치 | 변수 | 설명 | 필수 여부 |
|------|------|------|-----------|
| 1 | 학생명 | 한글 또는 영문 이름 | 필수 |
| 2 | 총점 | 게임에서 획득한 점수 (숫자) | 선택 |
| 3 | 결과 | `정답수/총문제수` 형식 (예: `7/10`) | 선택 |
| 4 | 모드 | `full` / `category` / `speed` | 선택 (기본값: `full`) |
| 5 | 카테고리 | 카테고리 모드일 때 해당 카테고리명 | 선택 |

**인수 없이 실행 시** 사용법 안내:
```
사용법: /quiz-teacher-register [학생명] [총점] [정답수/총문제수] [모드] [카테고리]
예시:
  /quiz-teacher-register 김민준 85 7/10 full
  /quiz-teacher-register 이수아 120 9/10 speed
  /quiz-teacher-register 박지호 72 6/10 category 한국사
  /quiz-teacher-register 김민준          ← 학생만 등록 (점수 없음)
```

시작 전 아래 형식으로 처리 내용을 표시하세요:
```
등록 대상: [학생명] | 점수: [총점 또는 "없음"] | 결과: [정답/총문제 또는 "없음"] | 모드: [모드]
```

---

## STEP 1 — students/ 디렉토리 및 기존 파일 확인

1. `students/` 디렉토리가 존재하는지 확인하세요.
   - 없으면 생성합니다.
2. `students/{학생명}.json` 파일이 존재하는지 확인하세요.
   - 존재하면 → **기존 학생: 점수 추가** 처리
   - 없으면 → **신규 학생: 파일 생성** 처리

```
--- 등록 유형 ---
유형    : [신규 등록] 또는 [기존 학생 — 점수 추가]
파일 경로: students/{학생명}.json
```

---

## STEP 2 — 데이터 유효성 검증

점수 데이터가 제공된 경우 아래를 검증하세요.

| 검증 항목 | 기준 |
|-----------|------|
| 총점 | 0 이상 양의 정수 |
| 정답수 | 0 이상, 총문제수 이하 |
| 총문제수 | 1 이상 양의 정수 |
| 모드 | `full`, `category`, `speed` 중 하나 |
| 카테고리 | `category` 모드일 때 `한국사`, `세계지리`, `과학`, `예술과 문화` 중 하나 |

검증 실패 시:
```
❌ 유효성 오류: (오류 내용)
조치: (올바른 입력 형식 안내)
```

---

## STEP 3 — 파일 작성

### 신규 학생 — 파일 생성

아래 형식으로 `students/{학생명}.json`을 생성하세요.

점수 데이터가 있으면 `scores` 배열에 1개 항목 포함, 없으면 빈 배열:

```json
{
  "name": "{학생명}",
  "registeredAt": "YYYY-MM-DD",
  "scores": [
    {
      "date": "YYYY-MM-DD",
      "totalScore": N,
      "correctAnswers": N,
      "totalQuestions": N,
      "accuracy": N,
      "avgTime": null,
      "longestStreak": null,
      "mode": "{모드}",
      "selectedCategory": "{카테고리 또는 null}",
      "categoryScores": {}
    }
  ]
}
```

### 기존 학생 — 점수 추가

기존 JSON 파일을 읽고 `scores` 배열에 새 항목을 **추가**합니다.
기존 데이터는 절대 덮어쓰지 마세요.

---

## STEP 4 — 결과 출력

```
--- 등록 완료 ---
학생명    : {학생명}
처리 유형 : [신규 등록] 또는 [점수 추가]
파일      : students/{학생명}.json
누적 기록 : N회

최신 점수 정보:
  날짜     : YYYY-MM-DD
  총점     : N점
  정답률   : N% (N / N)
  모드     : {모드}
```

점수 없이 학생만 등록한 경우:
```
--- 등록 완료 ---
학생명    : {학생명}
처리 유형 : 신규 학생 등록 (점수 없음)
파일      : students/{학생명}.json
안내      : /quiz-teacher-register {학생명} [점수] [결과] 로 점수를 입력할 수 있습니다.
```
