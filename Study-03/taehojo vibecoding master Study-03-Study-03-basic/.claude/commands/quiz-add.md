# 새 퀴즈 문제 추가

questions.js 파일을 읽고, 지정된 카테고리와 난이도로 새 퀴즈 문제를 생성하여 추가해 주세요.

## 인수 파싱

`$ARGUMENTS`를 공백으로 분리하여 첫 번째 값을 카테고리($1), 두 번째 값을 난이도($2)로 사용하세요.

- 두 값이 모두 있으면: 해당 카테고리와 난이도로 문제를 생성하세요.
- 카테고리만 있으면: 난이도는 `medium`으로 기본 설정하세요.
- 값이 없으면: 사용법을 안내하세요.
  ```
  사용법: /quiz-add [카테고리] [난이도]
  예시: /quiz-add 한국사 hard
       /quiz-add 세계지리 easy
       /quiz-add 과학 medium
       /quiz-add 예술과문화 hard
  ```

시작 전 첫 줄에 아래 형식으로 작업 정보를 표시하세요.

```
추가 대상: 카테고리=[카테고리명], 난이도=[난이도]
```

---

## Step 1 — questions.js 분석

questions.js를 읽고 다음을 파악하세요.

1. **현재 최대 id 값** — 새 문제의 id는 `최대 id + 1`로 설정합니다.
2. **지정된 카테고리의 기존 문제 목록** — 중복 문제를 피하기 위해 확인합니다.
3. **지정된 카테고리·난이도 조합의 문제 수** — 균형 참고용으로만 사용합니다.

결과를 아래 형식으로 출력하세요.

```
--- 현황 파악 ---
전체 문제 수     : N개
최대 id         : N
신규 id         : N
[카테고리] 문제  : N개 (easy N / medium N / hard N)
```

---

## Step 2 — 문제 생성

아래 규칙에 따라 새 문제를 생성하세요.

### 형식 규칙

- `id`: Step 1에서 계산한 신규 id
- `category`: 인수로 받은 카테고리명 (기존 카테고리와 동일한 문자열 사용)
- `difficulty`: 인수로 받은 난이도 (`"easy"` / `"medium"` / `"hard"`)
- `question`: 명확한 한 문장의 질문
- `options`: 4개의 보기 배열 (오해 없는 간결한 표현)
- `correctAnswer`: 0-based 인덱스 (0~3)
- `explanation`: 정답 이유와 핵심 근거를 포함한 한두 문장

### 난이도 기준

| 난이도 | 기준 |
|--------|------|
| easy   | 교과서 수준의 기본 지식, 널리 알려진 사실 |
| medium | 구체적인 연도·수치·명칭이 필요한 지식 |
| hard   | 세부 사항, 혼동하기 쉬운 개념, 심화 지식 |

### 검증 가이드라인 (CLAUDE.md 준수 필수)

문제 생성 전 아래 4가지를 반드시 확인하고, 각 항목의 통과 여부를 표시하세요.

```
--- 검증 체크리스트 ---
1. 정답이 하나뿐인가?
   → ✅ / ⚠️ (사유)
   · 다른 해석이 가능하면 문제에 조건을 명시할 것
     예) "면적 기준으로", "2024년 기준으로"

2. 최상급 표현에 기준이 있는가?
   → ✅ 해당 없음 / ✅ 기준 명시됨 / ⚠️ 기준 불명확 (수정 내용)
   · '가장', '최초', '최대', '최고', '최소', '최장', '처음', '제일' 포함 시
     측정 기준을 문제 안에 명시할 것

3. 시간과 범위가 명확한가?
   → ✅ / ⚠️ (수정 내용)
   · 변할 수 있는 정보(인구, 경제 순위 등)는 시점 명시
   · 지리적·분류적 범위 한정 필요 시 명시

4. 교차 검증했는가?
   → ✅ 확실 / ⚠️ 논란 가능성 (처리 방법)
   · 의심스러운 정보는 2개 이상 출처 기준 확인
   · 논란 있는 내용은 주류 학설·정설 기준으로 작성하고 explanation에 명시
```

검증 항목 중 하나라도 ⚠️이면, 문제를 수정한 뒤 재검증하여 모두 ✅가 된 최종본으로 추가하세요.

---

## Step 3 — questions.js 수정

검증을 통과한 문제를 questions.js 배열의 **맨 끝 `];` 바로 앞**에 추가하세요.

기존 문제와 동일한 들여쓰기(4칸 공백)와 쉼표 규칙을 유지하세요.

추가할 코드 블록 형식:

```js
    {
        id: N,
        category: "카테고리명",
        difficulty: "난이도",
        question: "문제 내용",
        options: ["보기1", "보기2", "보기3", "보기4"],
        correctAnswer: N,
        explanation: "정답 설명"
    }
```

---

## Step 4 — 결과 출력

추가 완료 후 아래 형식으로 최종 결과를 출력하세요.

```
--- 추가 완료 ---
id           : N
카테고리     : [카테고리명]
난이도       : [난이도]
문제         : (문제 내용)
정답         : ①/②/③/④ (정답 텍스트)
검증 결과    : ✅ 4개 항목 모두 통과
```
