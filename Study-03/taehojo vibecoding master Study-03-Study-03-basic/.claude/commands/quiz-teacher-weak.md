# 취약 영역 분석

학생별 또는 클래스 전체의 취약한 카테고리·난이도 조합을 찾아 보충 학습 우선순위를 제시합니다.

## 인수 파싱

`$ARGUMENTS`를 확인하세요.

| 인수 | 동작 |
|------|------|
| (없음) 또는 `all` | 전체 학생의 클래스 취약점 분석 |
| `{학생명}` | 특정 학생 1명의 취약점 분석 |
| `category` | 카테고리별 클래스 전체 정답률 히트맵 |
| `bottom N` | 하위 N명의 취약점 집중 분석 (예: `bottom 3`) |

시작 전 분석 범위를 표시하세요:
```
분석 범위: [전체 클래스 / {학생명} / 카테고리 히트맵 / 하위 N명]
```

---

## STEP 1 — 데이터 로드

해당 학생의 `students/{학생명}.json` 또는 `students/` 내 전체 파일을 읽으세요.

```
--- 데이터 로드 ---
분석 대상: N명 (또는 {학생명})
총 게임 기록: N회
```

데이터가 없으면:
```
❌ 분석할 데이터가 없습니다.
조치: /quiz-teacher-register 로 학생과 점수를 먼저 등록하세요.
```

---

## STEP 2 — 카테고리별 취약점 분석

### 개인 분석 모드

해당 학생의 모든 게임 기록에서 카테고리별 누적 정답률을 계산하세요.

```
  [{학생명}] 카테고리별 정답률

  카테고리      정답률     시각화 (최대 20칸)          판정
  ─────────────────────────────────────────────────────────
  한국사        N.N%   [████████████░░░░░░░░]   [양호/주의/취약]
  세계지리      N.N%   [████████░░░░░░░░░░░░]   [양호/주의/취약]
  과학          N.N%   [████████████████████]   [양호/주의/취약]
  예술과 문화   N.N%   [████░░░░░░░░░░░░░░░░]   [양호/주의/취약]
```

판정 기준:
- **양호**: 정답률 70% 이상
- **주의**: 정답률 50% 이상 70% 미만
- **취약**: 정답률 50% 미만

### 클래스 전체 분석 모드

전체 학생의 카테고리별 평균 정답률을 계산하세요.

## STEP 3 — 난이도별 취약점 분석

게임 기록에서 난이도별 정답률을 계산하세요.
(난이도별 데이터가 없으면 "데이터 부족"으로 표시)

```
  난이도별 정답률

  난이도    정답률     시각화                   판정
  ───────────────────────────────────────────────
  easy      N.N%   [████████████████████]   양호
  medium    N.N%   [████████████░░░░░░░░]   주의
  hard      N.N%   [████░░░░░░░░░░░░░░░░]   취약
```

## STEP 4 — 취약 문제 패턴 분석

`questions.js`를 읽고, 취약 카테고리에 속하는 문제들을 파악하세요.

```
  [{취약 카테고리}] 문제 목록 (총 N개)

  난이도   문제 수   대표 문제 예시
  ────────────────────────────────────────────
  easy     N개      "(문제 텍스트 앞 30자...)"
  medium   N개      "(문제 텍스트 앞 30자...)"
  hard     N개      "(문제 텍스트 앞 30자...)"
```

---

## STEP 5 — 보충 학습 우선순위

분석 결과를 종합하여 보충 학습 우선순위를 제시하세요.

### 개인 모드
```
========================================
  [{학생명}] 보충 학습 권장 우선순위
========================================

  1순위 — 즉시 집중 필요 (정답률 50% 미만)
    카테고리: {카테고리명} (N.N%)
    이 카테고리 문제 수: N개 (easy N / medium N / hard N)
    권장 학습법: {easy부터 순차 학습 / hard 집중 반복 등}

  2순위 — 보완 필요 (정답률 50~70%)
    카테고리: {카테고리명} (N.N%)
    ...

  현재 강점 (정답률 70% 이상)
    {카테고리명}: N.N% — 유지 권장
```

### 클래스 전체 모드
```
========================================
  클래스 전체 취약 영역 분석
========================================

  카테고리별 클래스 평균 정답률 (N명 기준)
  ┌──────────────┬─────────┬────────────────────┬────────┐
  │ 카테고리     │ 평균    │ 시각화             │ 판정   │
  ├──────────────┼─────────┼────────────────────┼────────┤
  │ 한국사       │ N.N%    │ ████████████░░░░   │ 양호   │
  │ 세계지리     │ N.N%    │ ████████░░░░░░░░   │ 주의   │
  │ 과학         │ N.N%    │ ████░░░░░░░░░░░░   │ 취약   │
  │ 예술과 문화  │ N.N%    │ ████████████████   │ 양호   │
  └──────────────┴─────────┴────────────────────┴────────┘

  카테고리별 취약 학생 명단
    {취약 카테고리}: {학생명1} (N.N%), {학생명2} (N.N%), ...

  클래스 전체 권장 학습 포인트
    1순위: {카테고리명} — 전체 N명 중 N명이 취약 판정
    2순위: {카테고리명} — 전체 N명 중 N명이 취약 판정
```
