# 퀴즈 정답 정확성 검증

questions.js 파일을 읽고, 모든 문제(또는 지정된 범위)의 정답 정확성을 검증해 주세요.

## 인수 파싱

`$ARGUMENTS`를 확인하세요.

- **카테고리가 지정된 경우**: 해당 카테고리의 문제만 검증하세요.
- **숫자 범위가 지정된 경우** (예: `10 20`): 해당 id 범위의 문제만 검증하세요.
- **값이 없으면**: 전체 문제를 검증하세요.

검증 시작 전 첫 줄에 아래 형식으로 범위를 표시하세요.

```
검증 범위: [전체] 또는 [카테고리: (카테고리명)] 또는 [id: N ~ N]
```

---

## Step 1 — questions.js 분석

questions.js를 읽고 검증 대상 문제를 파악하세요.

현황을 아래 형식으로 출력하세요.

```
--- 검증 현황 ---
전체 문제 수     : N개
검증 대상 문제 수 : N개
카테고리 목록    : (카테고리1 N개, 카테고리2 N개, ...)
```

---

## Step 2 — 문제별 정확성 검증

각 문제에 대해 아래 4가지 항목을 검증하세요.

### 검증 항목

**[A] 구조 검증** — 필수 필드 존재 여부
- `id`, `category`, `difficulty`, `question`, `options`, `correctAnswer`, `explanation` 필드가 모두 존재하는가?
- `options`가 정확히 4개인가?
- `correctAnswer`가 0~3 범위의 정수인가?

**[B] 정답 일치 검증** — correctAnswer와 explanation의 정합성
- `explanation`에 언급된 정답이 `options[correctAnswer]`와 일치하는가?
- 설명이 다른 보기를 정답으로 암시하지 않는가?

**[C] 정답 유일성 검증** — 오답 가능성 확인
- 다른 보기가 정답으로 오해될 여지가 없는가?
- 질문의 조건이 명확하여 정답이 하나로 좁혀지는가?

**[D] 사실 정확성 검증** — 알려진 사실과의 일치 여부
- 역사적 연도, 인물, 사건이 일반적으로 알려진 사실과 일치하는가?
- 최상급 표현(`가장`, `최초`, `최대` 등)에 측정 기준이 명시되어 있는가?
- 변동 가능한 정보(통계, 순위 등)에 시점이 명시되어 있는가?

### 출력 형식 (문제별)

문제별로 이상이 없으면 간략히, 이상이 있으면 상세히 출력하세요.

**정상인 경우:**
```
[ID N] ✅ 카테고리 / 난이도 — 이상 없음
```

**이상이 있는 경우:**
```
[ID N] ⚠️ 카테고리 / 난이도
문제    : (문제 내용)
정답    : options[correctAnswer] = "(정답 텍스트)"
이상 항목:
  [A] 구조: ✅ / ⚠️ (사유)
  [B] 일치: ✅ / ⚠️ (사유)
  [C] 유일: ✅ / ⚠️ (사유)
  [D] 사실: ✅ / ⚠️ (사유)
권장 조치: (수정 방향 또는 재검토 필요 내용)
```

---

## Step 3 — 카테고리별 요약

카테고리별로 이상 문제 수를 집계하세요.

```
--- 카테고리별 검증 결과 ---
카테고리     | 전체 | ✅ 정상 | ⚠️ 이상
-------------|------|---------|--------
한국사       |  N개 |    N개  |   N개
세계지리     |  N개 |    N개  |   N개
...
```

---

## Step 4 — 최종 요약

```
--- 검증 최종 요약 ---
검증 범위      : [전체] 또는 [카테고리/범위]
검증 문제 수   : N개
✅ 정상        : N개 (N%)
⚠️ 이상 발견  : N개 (N%)

이상 유형별 집계:
  [A] 구조 오류  : N개
  [B] 정답 불일치: N개
  [C] 정답 중복  : N개
  [D] 사실 오류  : N개

수정 권고 우선순위:
  1순위 (즉시 수정): id N, N, N  ← [B] 또는 [D] 이상
  2순위 (검토 권고): id N, N, N  ← [A] 또는 [C] 이상
```

이상 문제가 없으면 아래 메시지를 출력하세요.

```
🎉 모든 문제가 검증을 통과했습니다.
```
