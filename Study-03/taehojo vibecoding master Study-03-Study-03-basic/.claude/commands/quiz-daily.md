# 퀴즈 일일 관리 루틴

questions.js를 대상으로 아래 7단계를 순서대로 실행하세요.
각 단계는 이전 단계가 성공해야만 진행합니다.
단계가 실패하면 즉시 멈추고 오류를 보고한 뒤 이후 단계를 실행하지 마세요.

---

## 실행 헤더

시작 시 아래 형식으로 출력하세요.

```
========================================
  퀴즈 일일 관리 루틴
  실행 일시: YYYY-MM-DD
========================================
```

---

## STEP 1 — 파일 구조 파악

**목표**: questions.js의 데이터 구조를 읽고 이해한다.

questions.js를 읽고 다음을 확인하세요.

1. 최상위 변수명 (`quizQuestions` 등)
2. 각 문제 객체의 필드 목록: `id`, `category`, `difficulty`, `question`, `options`, `correctAnswer`, `explanation`
3. `options` 배열의 원소 수 (4개인지 확인)
4. `correctAnswer` 값의 타입과 범위 (0~3 정수인지 확인)

**검증 조건**:
- 파일이 존재하고 읽을 수 있어야 합니다.
- 배열이 비어 있지 않아야 합니다.
- 무작위로 3개 문제를 샘플링하여 위 필드가 모두 존재하는지 확인하세요.

**성공 출력**:
```
[STEP 1] ✅ 파일 구조 파악 완료
  변수명        : quizQuestions
  문제 객체 필드: id, category, difficulty, question, options, correctAnswer, explanation
  options 원소  : 4개
  correctAnswer : 0~3 정수
  샘플 검증     : id N, id N, id N → 필드 이상 없음
```

**실패 시 즉시 중단**:
```
[STEP 1] ❌ 실패 — 파일 구조 파악 불가
  원인: (구체적인 오류 내용)
  조치: (해결 방법)
  → 이후 단계를 모두 중단합니다.
```

---

## STEP 2 — 문제 수 및 분포 확인

**목표**: 전체 문제 수와 카테고리·난이도별 분포를 집계한다.

STEP 1에서 읽은 데이터를 기반으로 다음을 집계하세요.

1. 전체 문제 수
2. 카테고리별 문제 수
3. 카테고리 × 난이도 교차 집계
4. 현재 최대 id 값

**검증 조건**:
- 전체 문제 수가 1개 이상이어야 합니다.
- 모든 문제의 `id`가 양의 정수여야 합니다.
- 중복 id가 없어야 합니다. (중복 발견 시 STEP 2 실패 처리)

**성공 출력**:
```
[STEP 2] ✅ 문제 수 및 분포 확인 완료

  전체 문제 수 : N개
  최대 id      : N

  카테고리 × 난이도 분포
  ┌──────────────┬──────┬────────┬──────┬──────┐
  │ 카테고리     │ easy │ medium │ hard │ 합계 │
  ├──────────────┼──────┼────────┼──────┼──────┤
  │ 한국사       │    N │      N │    N │    N │
  │ 세계지리     │    N │      N │    N │    N │
  │ 과학         │    N │      N │    N │    N │
  │ 예술과 문화  │    N │      N │    N │    N │
  ├──────────────┼──────┼────────┼──────┼──────┤
  │ 합계         │    N │      N │    N │    N │
  └──────────────┴──────┴────────┴──────┴──────┘
```

**실패 시 즉시 중단**:
```
[STEP 2] ❌ 실패 — 분포 확인 불가
  원인: (예: 중복 id 발견 → id N, N)
  조치: (해결 방법)
  → 이후 단계를 모두 중단합니다.
```

---

## STEP 3 — 카테고리별 부족 파악

**목표**: 각 카테고리에서 부족한 난이도 조합을 찾아낸다.

STEP 2 집계 결과를 기반으로 아래 기준을 적용하세요.

| 기준 | 내용 |
|------|------|
| 권장 최소 문제 수 | 카테고리당 easy 3개 이상, medium 4개 이상, hard 3개 이상 |
| 부족 판정 | 권장 수 미달 조합을 "부족"으로 표시 |
| 우선순위 산정 | 부족 수가 많은 카테고리·난이도부터 높은 우선순위 부여 |

**검증 조건**:
- STEP 2가 성공한 경우에만 실행합니다.
- 집계 결과가 STEP 2와 일치해야 합니다 (재계산 후 비교).

**성공 출력**:
```
[STEP 3] ✅ 카테고리별 부족 파악 완료

  부족 항목 (우선순위 순)
  ┌──────────────┬────────┬──────┬──────┬────────────────────────┐
  │ 카테고리     │ 난이도 │ 현재 │ 권장 │ 부족 수                │
  ├──────────────┼────────┼──────┼──────┼────────────────────────┤
  │ (카테고리)   │ (난도) │    N │    N │ N개 부족 ← 추가 권장   │
  │ ...          │ ...    │  ... │  ... │ ...                    │
  └──────────────┴────────┴──────┴──────┴────────────────────────┘

  부족 없음 항목: (카테고리 / 난이도 목록)
```

부족 항목이 없으면:
```
[STEP 3] ✅ 카테고리별 부족 파악 완료
  모든 카테고리·난이도 조합이 권장 기준을 충족합니다.
```

**실패 시 즉시 중단**:
```
[STEP 3] ❌ 실패 — 부족 파악 불가
  원인: (구체적인 오류 내용)
  → 이후 단계를 모두 중단합니다.
```

## STEP 4 — 새 문제 추가 전 중복 체크

**목표**: STEP 3에서 파악한 "부족 1순위" 카테고리·난이도 조합에 추가할 새 문제 후보를 생성하고, 기존 문제와의 중복 여부를 검사한다.

### 4-1. 새 문제 후보 생성

STEP 3 결과를 바탕으로 부족 1순위 카테고리·난이도 조합에 맞는 새 문제 후보를 1개 생성하세요.  
부족 항목이 없으면 이 단계를 건너뛰고 아래 메시지를 출력하세요.

```
[STEP 4] ⏭️ 건너뜀 — 부족 항목 없음, 새 문제 추가 불필요
```

### 4-2. 중복 검사

생성한 후보 문제에 대해 다음을 검사하세요.

| 검사 항목 | 기준 |
|-----------|------|
| 질문 텍스트 | 기존 문제 `question` 필드와 80% 이상 유사한 문장이 없어야 함 |
| 핵심 키워드 | 같은 카테고리 내에서 동일한 정답(인물·연도·사건명 등)을 묻는 문제가 없어야 함 |
| 보기 배열 | 동일한 보기 4개가 순서와 무관하게 모두 일치하는 문제가 없어야 함 |

**검증 조건**:
- 위 3가지 검사를 모두 통과해야 STEP 5로 진행합니다.
- 중복이 발견되면 후보 문제를 폐기하고 새 후보를 1개 더 생성한 뒤 재검사하세요. (최대 3회 재시도)
- 3회 재시도 후에도 중복이면 STEP 4 실패로 처리합니다.

**성공 출력**:
```
[STEP 4] ✅ 중복 체크 완료

  추가 대상: 카테고리=[카테고리명], 난이도=[난이도]
  후보 문제 : "(문제 내용)"
  정답 보기 : options[N] = "(정답 텍스트)"

  중복 검사 결과
    질문 유사도 : ✅ 기존 문제와 80% 미만 유사
    핵심 키워드 : ✅ 동일 정답 문제 없음
    보기 배열   : ✅ 동일 보기 세트 없음
```

**실패 시 즉시 중단**:
```
[STEP 4] ❌ 실패 — 3회 재시도 후에도 중복 해소 불가
  원인: (중복되는 기존 문제 id 및 내용)
  → 이후 단계를 모두 중단합니다.
```

## STEP 5 — 문제 추가 후 형식 검증

**목표**: STEP 4에서 확정된 문제를 questions.js에 추가하고, 추가 후 파일 전체의 형식을 검증한다.

STEP 4가 "건너뜀"이면 이 단계도 건너뜁니다.

### 5-1. 문제 추가 전 최종 검증 체크리스트

추가 전에 아래 4가지를 반드시 확인하세요.

```
--- 추가 전 검증 체크리스트 ---
1. 정답이 하나뿐인가?
   → ✅ / ⚠️ (사유 및 수정 내용)

2. 최상급 표현에 측정 기준이 명시되어 있는가?
   → ✅ 해당 없음 / ✅ 기준 명시됨 / ⚠️ 기준 불명확 (수정 내용)

3. 시간과 범위가 명확한가?
   → ✅ / ⚠️ (수정 내용)

4. 교차 검증했는가?
   → ✅ 확실 / ⚠️ 논란 가능성 (처리 방법)
```

하나라도 ⚠️이면 수정 후 재검증하여 모두 ✅가 된 최종본만 추가하세요.

### 5-2. questions.js에 추가

검증을 통과한 문제를 배열의 **맨 끝 `];` 바로 앞**에 추가하세요.

- `id`: 현재 최대 id + 1
- 들여쓰기: 4칸 공백
- 쉼표 규칙: 기존 패턴과 동일하게 유지

### 5-3. 추가 후 형식 검증

추가 후 questions.js를 다시 읽어 전체 문제의 형식을 검증하세요.

| 검증 항목 | 기준 |
|-----------|------|
| 필드 완전성 | 모든 문제에 7개 필드(`id`, `category`, `difficulty`, `question`, `options`, `correctAnswer`, `explanation`) 존재 |
| id 연속성 | 중복 id 없음 |
| options 길이 | 모든 문제에서 4개 |
| correctAnswer 범위 | 0~3 정수 |
| category 값 | 기존 카테고리 문자열과 일치 |
| difficulty 값 | `"easy"`, `"medium"`, `"hard"` 중 하나 |

**성공 출력**:
```
[STEP 5] ✅ 문제 추가 및 형식 검증 완료

  추가된 문제
    id          : N
    카테고리    : (카테고리명)
    난이도      : (난이도)
    문제        : "(문제 내용)"
    정답        : options[N] = "(정답 텍스트)"
    검증 결과   : ✅ 체크리스트 4개 항목 모두 통과

  형식 검증 결과 (전체 N개 문제)
    필드 완전성    : ✅
    id 중복 없음   : ✅
    options 길이   : ✅
    correctAnswer  : ✅
    category 값    : ✅
    difficulty 값  : ✅
```

**실패 시 즉시 중단**:
```
[STEP 5] ❌ 실패 — 형식 검증 오류
  원인: (오류가 발생한 문제 id 및 필드명, 구체적인 내용)
  조치: questions.js를 수동으로 확인하고 오류를 수정하세요.
  → 이후 단계를 모두 중단합니다.
```

## STEP 6 — 전체 데이터 백업

**목표**: questions.js의 현재 상태를 백업 파일로 저장한다.

백업 파일명 형식: `questions.backup.YYYYMMDD.js`  
저장 위치: questions.js와 동일한 디렉토리

백업 파일 내용: questions.js의 전체 내용을 그대로 복사합니다.  
파일 첫 줄에 아래 주석을 추가하세요.

```js
// 백업 일시: YYYY-MM-DD | 문제 수: N개 | 원본: questions.js
```

**검증 조건**:
- 백업 파일이 실제로 생성되어야 합니다.
- 백업 파일의 줄 수가 원본과 일치해야 합니다 (주석 1줄 추가 반영).
- 백업 파일 이름에 오늘 날짜가 올바르게 포함되어야 합니다.

**성공 출력**:
```
[STEP 6] ✅ 백업 완료

  백업 파일 : questions.backup.YYYYMMDD.js
  문제 수   : N개
  원본 크기 : N줄
  백업 크기 : N줄 (주석 1줄 포함)
```

**실패 시 즉시 중단**:
```
[STEP 6] ❌ 실패 — 백업 생성 불가
  원인: (구체적인 오류 내용)
  조치: 디렉토리 쓰기 권한을 확인하세요.
  → 이후 단계를 모두 중단합니다.
```

## STEP 7 — 실행 결과 상세 보고

**목표**: 모든 단계의 실행 결과를 종합하여 최종 보고서를 출력한다.

### 단계별 실행 결과 요약

```
========================================
  퀴즈 일일 관리 루틴 — 최종 보고서
  실행 일시: YYYY-MM-DD
========================================

  단계별 결과
  ┌────────┬──────────────────────────┬────────┐
  │ 단계   │ 내용                     │ 결과   │
  ├────────┼──────────────────────────┼────────┤
  │ STEP 1 │ 파일 구조 파악           │ ✅/❌  │
  │ STEP 2 │ 문제 수 및 분포 확인     │ ✅/❌  │
  │ STEP 3 │ 카테고리별 부족 파악     │ ✅/❌  │
  │ STEP 4 │ 중복 체크                │ ✅/⏭️/❌ │
  │ STEP 5 │ 문제 추가 및 형식 검증   │ ✅/⏭️/❌ │
  │ STEP 6 │ 전체 데이터 백업         │ ✅/❌  │
  └────────┴──────────────────────────┴────────┘
```

### 데이터 변화 요약

```
  데이터 변화
    루틴 전 문제 수 : N개
    루틴 후 문제 수 : N개 (+ N개 추가 / 변화 없음)
    백업 파일       : questions.backup.YYYYMMDD.js
```

### 카테고리 현황 (루틴 후 기준)

```
  카테고리 현황 (루틴 후)
  ┌──────────────┬──────┬────────┬──────┬──────┬───────────────────┐
  │ 카테고리     │ easy │ medium │ hard │ 합계 │ 상태              │
  ├──────────────┼──────┼────────┼──────┼──────┼───────────────────┤
  │ 한국사       │    N │      N │    N │    N │ ✅ 충족 / ⚠️ 부족 │
  │ 세계지리     │    N │      N │    N │    N │ ✅ 충족 / ⚠️ 부족 │
  │ 과학         │    N │      N │    N │    N │ ✅ 충족 / ⚠️ 부족 │
  │ 예술과 문화  │    N │      N │    N │    N │ ✅ 충족 / ⚠️ 부족 │
  └──────────────┴──────┴────────┴──────┴──────┴───────────────────┘
```

### 다음 실행 시 권장 작업

- 여전히 부족한 카테고리·난이도 조합이 있으면 목록으로 출력하세요.
- 부족 항목이 없으면 아래를 출력하세요.

```
  모든 카테고리·난이도 조합이 권장 기준을 충족합니다.
  다음 실행 시에도 동일한 루틴을 진행합니다.
```

### 최종 상태

모든 단계가 성공하면:
```
========================================
  🎉 루틴 완료 — 모든 단계 성공
========================================
```

중간에 실패한 단계가 있으면:
```
========================================
  ⛔ 루틴 중단 — STEP N에서 오류 발생
  원인: (오류 요약)
  조치: (권장 해결 방법)
========================================
```
