# 퀴즈 범위 검토

questions.js 파일을 읽고, 지정된 번호 범위의 문제를 검토하여 난이도 분포와 정답 분포를 분석해 주세요.

## 검토 범위 파싱

`$ARGUMENTS`를 공백으로 분리하여 첫 번째 값을 시작 번호($1), 두 번째 값을 끝 번호($2)로 사용하세요.

- 두 값이 모두 있으면: id가 $1 이상 $2 이하인 문제만 검토하세요.
- 값이 하나만 있으면: 해당 번호의 문제 1개만 검토하세요.
- 값이 없으면: 사용법을 안내하세요. (`사용법: /quiz-range [시작번호] [끝번호]`)

검토 시작 전 첫 줄에 아래 형식으로 범위를 표시하세요.

```
검토 범위: [$1번 ~ $2번] (총 N개 문제)
```

## 문제 목록 출력

범위 내 각 문제를 아래 형식으로 나열하세요.

```
[문제 ID] 카테고리 / 난이도
문제: (문제 내용)
정답: (정답 텍스트 또는 보기 번호)
```

---

## 난이도 분포 분석

범위 내 문제의 난이도(difficulty 또는 level 필드) 분포를 집계하세요.

출력 형식:

```
--- 난이도 분포 ---
쉬움(easy)   : N개 (N%)
보통(medium) : N개 (N%)
어려움(hard) : N개 (N%)
```

막대 그래프로 시각화하세요. (■ 1개 = 1문제, 최대 20칸)

```
쉬움   | ■■■■■ 5
보통   | ■■■■■■■■■■ 10
어려움 | ■■■ 3
```

난이도 필드가 없거나 값이 다양한 경우, 실제 필드값 기준으로 집계하세요.

---

## 정답 분포 분석

객관식 문제의 경우 정답 보기 번호(answer 필드) 분포를 집계하세요.

출력 형식:

```
--- 정답 분포 ---
1번 정답: N개 (N%)
2번 정답: N개 (N%)
3번 정답: N개 (N%)
4번 정답: N개 (N%)
```

막대 그래프로 시각화하세요. (■ 1개 = 1문제, 최대 20칸)

```
1번 | ■■■■ 4
2번 | ■■■■■■■ 7
3번 | ■■■■■ 5
4번 | ■■■■■■ 6
```

정답이 특정 번호에 편중된 경우(한 번호가 40% 이상) 아래 경고를 표시하세요.

```
⚠️ 정답 편중 감지: N번 정답 비율이 N%로 높습니다. 정답 분포를 재검토하세요.
```

---

## 최종 요약

```
--- 검토 요약 ---
검토 범위    : $1번 ~ $2번
검토 문제 수 : N개
난이도 분포  : 쉬움 N / 보통 N / 어려움 N
정답 분포    : 1번 N / 2번 N / 3번 N / 4번 N
편중 경고    : 있음 / 없음
```
